# Environment Configuration

Copy this file to `.env` and adjust the values according to your environment:

```bash
cp .env.template .env
```

## Required Directories

The application will automatically create the following directories if they don't exist:
- `DOWNLOAD_TEMP_DIR`: Temporary storage for downloads in progress
- `DOWNLOAD_DEST_DIR`: Final destination for completed downloads

Make sure the application has write permissions for these directories.

## Configuration Variables

### Server
- `PORT`: HTTP server port (default: 3000)
- `URL`: Public URL of the server (default: http://localhost:3000)

### CORS
- `ALLOWED_ORIGINS`: Comma-separated list of allowed origins for CORS

### Downloads
- `DOWNLOAD_TEMP_DIR`: Temp directory for active downloads (default: /tmp/downloads)
- `DOWNLOAD_DEST_DIR`: Final destination directory (default: /home/downloads)
- `MIN_STORAGE_GB`: Minimum GB required before starting downloads (default: 5)
- `MAX_PENDING_DOWNLOADS`: Maximum simultaneous pending downloads (default: 10)
- `DOWNLOAD_TIMEOUT_MINUTES`: Timeout for downloads (default: 60)

### Maintenance
- `CLEANUP_RETENTION_DAYS`: Days to keep orphaned files (default: 7)

### Events
- `EVENT_BUFFER_SIZE`: SSE circular buffer size (default: 100)
- `PROGRESS_THROTTLE_MS`: Throttle time for progress events (default: 500)
